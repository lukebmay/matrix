# Markdown

## Lists

- Numbered: always `1.` (renders fine; easy reorder).
- Unordered: `-` (not `*`).

## Headings

- Hierarchy: `#` then `##`, `###`, … — no skipped levels.
- No trailing colons in headings.

## Code

- Fenced blocks with a language tag when possible.
- Avoid indented (4-space) code blocks.

## Links

- Descriptive text; raw URLs only when the URL itself should be readable.
- Prefer relative links inside the repo.

## Format

- Soft wrap ~80–100 cols for diffs/editors.
- `**bold**` / `*italic*` (not `__` / `_` alone for those roles).
- `---` sparingly.
- Tasks: `- [ ]` / `- [x]`.
